ITIL Overview

This folder contains ITIL concepts and how they map directly to cloud security, cloud governance, FinOps, and multi-cloud operations.

Purpose:
- Demonstrate operational maturity
- Show alignment with enterprise processes
- Connect ITIL concepts to cloud environments (AWS, Azure, GCP)
- Strengthen readiness for governance, FinOps, and cloud security roles

Core ITIL Domains Covered:
1. Incident Management
2. Change Management
3. Problem Management
4. Configuration Management
5. Service Catalog
6. Service Continuity
7. Cloud Governance Mapping
