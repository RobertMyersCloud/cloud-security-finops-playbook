# Cloud Governance Maturity Model (Level 1 → Level 5)

Defines how mature cloud governance processes are.

---

# Level 1 — Ad Hoc
- No standards  
- Manual provisioning  
- Inconsistent IAM  
- Reactive security  

---

# Level 2 — Basic Controls
- Tagging required  
- Basic IAM rules  
- Budgets set  
- Manual reviews  

---

# Level 3 — Standardized Governance
- Landing Zones  
- Guardrails  
- Policy-as-Code  
- Standard images  
- Naming conventions  

---

# Level 4 — Automated Governance
- Auto enforcement  
- CI/CD policy gates  
- Auto-remediation  
- Continuous compliance  

---

# Level 5 — Optimized Governance
- Real-time risk scoring  
- Zero Trust enforcement  
- Policy drift autocompliance  
- Full FinOps integration  
- Governance dashboards  

---

# Summary
The maturity model shows where an organization is and what’s required to reach enterprise-level governance.
